# STEPFlow V1

## INSTRUCCIONES PRINCIPALES

Antes de ejecutar cualquier acción dentro de este proyecto:

1. Leer completamente el archivo `JORDAN_CORE.md`
2. Entender que el objetivo principal es:
   - generar clientes
   - aumentar conversiones
   - escalar el sistema

---

## REGLAS DE OPERACIÓN

- No crear estructuras nuevas si ya existe una compatible
- Priorizar siempre:
  - captación de leads
  - envío de mensajes
  - conversión a demo
  - cierre de venta
- Usar las skills disponibles cuando sea necesario
- Mantener lógica de sistema (no tareas aisladas)

---

## SEGURIDAD

- Nunca hardcodear credenciales
- Nunca exponer API keys
- Usar variables de entorno (.env)
- Separar:
  - datos de demo
  - datos de clientes
  - lógica del sistema

---

## ARQUITECTURA

Trabajar siempre bajo esta estructura:

- JORDAN_CORE.md → cerebro del sistema
- skills/ → funcionalidades específicas
- docs/ → documentación
- data/ → almacenamiento de datos
- logs/ → registro de actividad

---

## OBJETIVO ACTUAL (V1)

Construir sistema mínimo funcional que logre:

1. Captar contactos
2. Clasificar nicho
3. Enviar primer mensaje
4. Registrar interacción
5. Detectar respuesta
6. Clasificar lead
7. Llevar a demo
8. Preparar cierre de venta

---

## COMPORTAMIENTO ESPERADO

- Pensar como sistema, no como asistente
- Priorizar impacto en ventas
- Reducir complejidad innecesaria
- Proponer mejoras constantemente

---

## IMPORTANTE

Si hay dudas:
- analizar contexto existente
- consultar antes de ejecutar acciones críticas
